# BDR 시안 의뢰 — 대회 "기록실" 탭 (전체 선수 스탯)

> 디자인 클로드(BDR v2.X 프로젝트)용 의뢰서. NBA.com/stats 참고. **백엔드 실측 완료 — 신규 DB 0.**

## 1. 컨텍스트
- 대상: 대회 상세 `/(web)/tournaments/[id]` — **신규 탭 "기록실"** 추가.
- 현재 탭: 대회소개 · 경기일정 · 대진표 · 참가팀 · 규정 (`tournament-tabs.tsx`). → 사이 또는 끝에 "기록실" 삽입(아이콘 `leaderboard`).
- 우선순위 P1 · 박제 등급 D(미존재) → A.
- 목적: 대회에 참여한 **전체 선수의 스탯**을 한 화면에서 정렬·비교(NBA.com 리그 리더 테이블 경험).

## 2. 변경 요구사항
### 2-1. 핵심
- "기록실" 탭 = **정렬 가능한 전체 선수 스탯 테이블** + 상단 리더 하이라이트.
- **평균(Per Game) ↔ 합계(Total) 토글** (NBA.com 의 PerGame/Totals).
### 2-2. 신규 컴포넌트
- (a) **리더 카드 줄** — 득점/리바운드/어시스트 TOP (기존 `tournament-stat-leaders-card.tsx` 재사용/확장).
- (b) **스탯 테이블** — sticky 선수명 열 + 정렬 헤더 + 가로 스크롤. 컬럼: 선수 · 팀 · GP · MIN · PTS · REB(OREB/DREB/TREB) · AST · STL · BLK · TOV · PF · FG% · 3P% · FT% · eFG% · TS% · +/- · EFF.
- (c) **필터 바** — 종별/디비전 · 팀 · 검색(선수명). 정렬 기본 = PTS desc.
### 2-3. 보존(변경 금지)
- AppNav frozen · 기존 5탭 · 디자인 토큰 · 기존 리더 카드의 데이터 계약.

## 3. 디자인 명세
### 3-1. 레이아웃
- 데스크톱: 리더 카드 3장 → 필터 바 → 와이드 sortable 테이블(가로 스크롤). NBA.com 처럼 헤더 클릭 정렬(▲▼), 상위 1위 셀 강조.
- 모바일(≤720px): 첫 열(선수명+팀) **sticky** + 나머지 가로 스크롤. 또는 "핵심 5스탯 카드 + 더보기" 폴백. iOS input 16px / 행 탭 44px.
### 3-2. 컴포넌트

### 3-3. Interaction states

### 3-4. Connections
- 진입: 대회 상세 탭. 다음: 선수행 → `/users/[id]`(클레임) / 팀 → `/teams/[id]`. 복귀: 대회 상세.

## 5. ★ 백엔드 실측 — 데이터 출처 (신규 DB 0)
- **소스**: `MatchPlayerStat`(경기별 풀 박스스코어) — points/3PM·3PA/FGM·FGA/FTM·FTA/OREB·DREB·TREB/assists/steals/blocks/turnovers/personal_fouls/plus_minus/minutes/efficiency/TS%/eFG% 전부 보유.
- **대회 집계**: `MatchPlayerStat.groupBy(tournamentTeamPlayerId, _sum)` (대회 완료 매치 필터) — 이미 `src/lib/tournaments/stat-leaders.ts` 가 이 패턴(현재 4부문 TOP) → **전 컬럼·전 선수로 확장만**.
- **기존 API**: `/api/v1/tournaments/[id]/player-stats` 존재 → 웹용 엔드포인트는 동일 집계 재사용.
- **Per Game**: `_sum / GP`(선수별 출전 매치 수). Total/PerGame 둘 다 산출 가능.
- **선수 식별**: `TournamentTeamPlayer.userId`(클레임) → 개인 페이지 링크 / `player_name`(미클레임) → 링크 없이 이름만.
- 표시할 수 없는 스탯 없음 — NBA.com 컬럼 대부분 실현 가능. (단 NBA의 DD2/TD3/FP 같은 파생 지표는 선택 — 필요 시 계산 가능.)
