## 1. 디자인시스템 (관리자 Toss — 기존 정의 사용)
`toss.css` 그대로: 라이트(배경 #F2F4F6 / 카드 #FFFFFF), Toss Blue #3182F6, **카드 라운딩 16~24px**, 입력 14px, Pretendard, lucide. 의미색 ok #15B86A / warn #FF9500 / danger #F04452. 컴포넌트 `.ts-card/.ts-btn/.ts-input/.ts-badge/.ts-chip/.ts-segment`. mybdr 프론트(BDR Red·다크) 아님.

## 2. 컨텍스트 / 왜 재설계
- 현재 mybdr 새 대회 만들기 = **이중 네비(admin 사이드바 + 대회운영 서브탭) + 4개 모드 탭(Quick/단계별/PDF/협회) + 좁은 1컬럼 긴 스크롤** → 산만·결정피로·미완결(생성 후 SetupHub 8카드).
- 목표 = BDR-join-v1처럼 **한 화면 3컬럼에서 스캔·입력·생성 완결**.

## 3. 레이아웃 — 단일 페이지 3컬럼 (핵심)
- **focused 작업 모드**: 생성 화면에선 상단 대회운영 서브탭 숨김(풀폭). 좌측 admin 사이드바는 유지하되 본문을 넓게. (헤더: "새 대회 만들기" + 서브텍스트 + 우상단 [이전 대회 불러오기] · [PDF로 채우기] · [협회 마법사] 보조 버튼.)
- **3컬럼** (데스크톱 ≥1024px):
  - **Col 1 — 기본 정보** (`.ts-card`, 아이콘 헤더): 대회명*(필수) / 대회 상태(드롭다운) / 대회 성별(혼성·남·여) / 시작·종료일 / (선택)소속 시리즈.
  - **Col 2 — 경기장 + 상세**: 경기장(직접 입력 + [검색]) · 등록 경기장 리스트(빈 상태 "등록된 경기장이 없습니다") / 상세(주최·주관·후원 · 경기시간·경기구·경기방식 = 기존 프리셋 입력 · 포스터/링크).
  - **Col 3 — 종별·디비전 구성**: 우상단 [+] → DivisionGenerator(종별 마스터 admin_categories 연동: 성별→종별→디비전). 추가된 종별 카드 = 종별명 인라인 + 디비전별 정원/참가비 인라인. 빈 상태 "우측 상단 +로 종별을 추가하세요".
- **하단 고정 바**(sticky): 좌 [취소] / 우 큰 primary [대회 생성 완료]. 항상 노출.
- **모바일(≤720px)**: 3컬럼 → 1열 stack(기본→경기장→상세→종별 순), 하단 고정 바 유지, 입력 16px·버튼 44px.

## 4. 모드 탭 → 보조 진입점 (단순화)
- Quick 탭 제거: 3컬럼 폼 자체가 "이름만 필수, 나머지 선택"이라 이미 빠름.
- PDF 요강·협회 대회 = **우상단 보조 버튼**으로(메인 흐름에서 빼기). 협회 마법사는 super_admin만 노출.

## 5. Interaction / States
- empty(경기장 0 / 종별 0) · validation(대회명 미입력 시 생성 버튼 비활성 + 인라인) · loading(생성 중) · success(생성 후 이동) · "이전 대회 불러오기" 모달(기존 대회 검색→프리필, format/settings·종별 포함).
- 종별 추가 시 "저장하면 종별 규칙 N건 자동 생성" 안내(배너).

## 6. Connections
- 진입: 대회 관리 목록 [+ 새 대회] / 대시보드 바로가기.
- 생성 완료 → 대회 상세(참가팀+대진 통합) 또는 SetupHub 잔여(공개·기록자만).
- 취소 → 목록.

## 7. 데이터 (참고 — 신규 없음)
기존 `Tournament`(name/status/gender/startDate·endDate/places/organizer·host·sponsors/gameTime·Ball·Method/poster) + Track B(admin_categories·DivisionRule 자동생성·이전대회 복사). **API/스키마 변경 없음** — 레이아웃·IA·디자인 재구성.
