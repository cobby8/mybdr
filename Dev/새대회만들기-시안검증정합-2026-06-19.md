## 3. 🔴 백엔드 정합 (CLI 오판 방지 — 대부분 기존 모델 매핑)
시안 mock state → mybdr 실제(실측):

| 시안 | mybdr 실제 | 조치 |
|------|------|------|
| 정규대회(isRegular/seriesId/seriesName) | **`tournament_series` + `Tournament.series_id` 존재** | 매핑. 신규 정규대회 생성 API만 보강 |
| 장소·코트(venues[].courtCount/naming) | **`courts`/`court_infos` 존재** | 매핑. 코트 수·명명(숫자/알파벳)·다중 코트 표현 정합 |
| dates[]↔courtIds[] (날짜별 코트 N:N) | 미존재(추정) | **신규 관계 필요**(대회일정↔코트). schema 검토 후 추가 |
| categories.divisions.{method,cap,fee,dateId,courtId} | `TournamentDivisionRule`(format/settings/feeKrw 존재) + cap·**날짜·코트** | Track B 확장: dateId/courtId 매칭 추가 |
| 경기 설정(유니폼/방식/파울/타임아웃) | **`Tournament.game_rules Json` 존재** + `game_time/ball/method` | **기존 game_rules JSON 형태에 정합**(신규 컬럼 금지) |
| 게시 모달(결제수단/계좌) | `Tournament` bank/entry_fee 필드 존재 | 매핑. PG는 '준비 중'(계좌이체만) |
| admin_categories(종별 마스터) | 신규(Track B) | Track B대로 |

→ **신규 생성은 admin_categories + 날짜↔코트 배정 관계 정도**. courts/series/game_rules는 **재사용**(중복 생성 금지).
