## 1. 디자인시스템 (Toss 스타일)

`toss.css` — mybdr 관리자 전 화면 공통 토큰·컴포넌트 키트. BDR-join-v1 `globals.css` 기준.

- 폰트 **Pretendard**, 아이콘 **lucide** (이 모듈 한정 — CDN `unpkg.com/lucide`)
- Toss Grey scale(50~900), 부드러운 그림자, 넉넉한 여백
- 컴포넌트: 버튼(primary/secondary/ghost/danger) · 카드 · 입력 · 뱃지 · 칩 · 세그먼트 · 토글 · 체크 · 스텝닷 · 사이드바 · 테이블 · 모달 · 토스트

### ⭐ 관리자 콘솔 리스킨 (AdminConsole.html) — 우선 5화면
기존 admin 기능 1:1 유지 + Toss 재스킨. 상태탭·검색/필터·정렬·선택·페이지네이션·상세 슬라이드인·mock 토글 동작.
- **사용자** 등급/상태/신고 · **대회** 접수현황(운영상세는 Phase 4) · **경기** 매치/픽업 · **커뮤니티** 신고/숨김 · **단체** 승인대기
- 나머지(대시보드·결제·코트·팀·설정)는 Phase 2에서 리스킨 완료 (아래).

### 관리자 콘솔 Phase 2 (AdminConsole.html)
- **대시보드** — KPI 4종 + 활동 추이 막대 차트 + 최근 활동 로그(심각도 점)
- **결제** — 매출/대기/실패 KPI + 거래 테이블(완료/대기/환불/실패) + 환불 처리
- **코트** — 운영중/승인대기/중단, 무료/유료, 신규 승인
- **팀** — 인원·소속단체·대회 참가
- **설정** — 세그먼트 탭(종별 마스터 / 일반 / 대표자), 종별 마스터를 설정 안에 통합

### ⭐ 참가신청서 3단계 (사용자용)
- **Step 1 참가팀 선택 + 정보 확인** — 로그인 사용자의 가입 팀에서 선택(여러 개 가능, 1개면 자동). 선택 즉시 팀명·대표자·연락처·지역·유니폼 확인. 비밀번호 입력 없음(세션).
- **Step 2 종별·디비전** — 종별→디비전 버튼. 디비전별 **현재/정원** 표시, 초과 시 **대기접수 뱃지(주황)**.
- **Step 3 출전 선수 선택** — 팀 로스터(기존 가입 선수)에서 **체크박스 선택**. 명단 입력 폐지. 선수 정보는 표시만.
- **유니폼 단계 폐지** (팀 페이지 설정값 표시만). 완료 = 입금 안내 / 정원초과 시 "대기 접수하기".
- ⏳ **출전 최소인원 가드 · 게스트 추가** = 추후 결정 → 코드에 `MIN_PLAYERS_GUARD` / `ALLOW_GUEST` 토글 + 안내 문구로 자리만 마련.

### ⭐ 종별 마스터 (admin_categories 복원)
- 종별 CRUD + `divisions`·`ages` 태그 입력(Enter 추가 / X 제거).
- **4종 실데이터 시드 복원**: 일반부(D3~D8) / 유청소년(하모니·i1~i4, 연령 U8~U18) / 대학부(U1~U3) / 시니어(S1~S3, 연령 +40~+70).

### 사이드바 4메뉴 IA
- 운영(대시보드·대회관리·참가팀관리) / 설정(종별마스터) / 미리보기(참가신청서). 대시보드·대회관리·참가팀은 **Phase 2** (P2 뱃지).
