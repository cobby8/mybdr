## 2. 의뢰 예시 (Phase 13 마이페이지)

### Example 1: 마이페이지 hub (실제 의뢰)

```
# BDR 시안 의뢰 — Phase 13 마이페이지 허브

## 1. 컨텍스트
- 의뢰 대상: /profile 페이지 (BDR v2.3 갱신)
- 우선순위: P0 (사용자 직접 보고)
- 박제 등급 변화: B → A (디테일 보강 + 마이페이지 hub 통합)
- 시급도: 이번 주

## 2. 변경 요구사항

### 2-1. 핵심 변경
- /profile 의 main 영역 하단에 마이페이지 hub 섹션 추가
- 16개 흩어진 /profile/* 서브페이지를 5 그룹 + 3단계 위계 카드 그리드로 통합

### 2-2. 추가 / 신규
- Tier 1 큰 카드 4종 (프로필 / 내 농구 / 내 성장 / 내 활동)
- Tier 2 중간 카드 4종 (예약 / 리포트 / 알림 / 배지)
- Tier 3 작은 카드 2종 (설정 / 결제)
- 보조 정보 (다음 경기 D-N / 소속 팀 / 최근 활동)

### 2-3. 보존
- AppNav 그대로 (03 frozen)
- HeroCard / 시즌 스탯 / 다가오는 일정 / 최근 활동 그대로
- 소속 팀 / 활동 뱃지 카드 그대로

## 3. 디자인 명세

### 3-1. 레이아웃
- 데스크톱: 좌 320px sticky aside (HeroCard + 소속팀 + 뱃지) + 우측 main
  - main 상단: 시즌 스탯 6 + 다가오는 일정 + 최근 활동 (기존)
  - main 하단: 마이페이지 hub 5 카드 (3-2 grid 또는 4-4-2 위계)
- 모바일: 1열 stack — HeroCard → 시즌 스탯 → 일정 → 활동 → hub 5 카드

### 3-2. 컴포넌트
- mypage-hub-card (3 사이즈: lg / md / sm)
- 카드 = 큰 이모지 + 그룹 제목 + 4 항목 + "전체 보기 →"

### 3-3. States
- 알림 카드: 미확인 N건 뱃지
- 본인인증 카드: ✓ 뱃지 (Phase 12 활성화 후) / "미인증" (그 전)

### 3-4. Connections
- 진입: 헤더 user dropdown / 더보기 "마이페이지" / /profile 직진입
- 카드 클릭: 해당 /profile/* 페이지
- 복귀: AppNav / PageBackButton

## 4. 박제 룰
- 모두 자동 적용 (위 §1 룰 4종)

## 5. 산출물
- Dev/design/BDR v2.3/screens/Profile.jsx 갱신
- (선택) Dev/design/BDR v2.3/screens/Refund.jsx 신규
- components.jsx moreGroups "계정·도움" 그룹 첫 항목 "마이페이지" 추가

## 6. 추가 컨텍스트

- 사용자 직접 의견: "마이페이지가 없음. MyBDR 핵심인데 프로필/개인정보/농구정보 통합 필요"
```

## 4. 자주 묻는 질문 (FAQ)

### Q1. AppNav 외에 다른 frozen 컴포넌트는?

현재 frozen 은 AppNav 만. 다음은 frozen 후보 (사용자 결정에 따라):
- 다크/라이트 토글 (ThemeSwitch) — AppNav 안에 포함
- HeroCard (사용자 결정 §3 — 레이팅 stat 노출 금지)
- ProfileSettings 섹션 nav (이모지 아이콘 §4-2)

→ 시안 작업 시 위 컴포넌트 수정 시 PM 확인 필수.

### Q2. 새 라우트 추가해도 되는가?

신규 라우트 = 사용자 결정 §1-4 (메인 탭 9개) 변경과 직결. 다음만 OK:
- /referee-info (Phase 11 신규 — refereeInfo 옵션 B)
- 새 시안 페이지 (예: 마이페이지 hub 는 /profile 그대로)

신규 메인 탭 추가 (10번째 탭 등) 는 PM 결정 필수.

### Q3. 시안 외 영역 (admin / referee 등) 도 의뢰?

E 등급은 박제 대상 X. 단 사용자가 "/admin 의 통계 페이지 디자인 새로" 같은 자체 디자인 의뢰는 가능. 이때:
- AppNav X (별도 셸 사용)
- 디자인 토큰만 일치
- 사용자 결정 §1 (헤더) 적용 X

### Q4. PM 누구?

운영 의사결정 = 사용자 (수빈) 또는 원영 (Co-PM, main 머지 담당).
디자인 의사결정 = 사용자 (수빈).

### Q5. 시안의 Korean labels 톤은?

- 격식 / 존댓말 (말풍선 / CTA)
- 짧고 명확 (5단어 이내)
- "예: " "ex)" 시작 placeholder 금지 (사용자 결정 §B)
- 영문 라벨은 카피 보조 (eyebrow / display 폰트 등)
