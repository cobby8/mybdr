# 대회 생성 위저드 — 종별 / 디비전 / 연령설정 규칙

> **디자인 코워크 전달용** · 2026-06-28 작성 · 운영 코드 + DB 실측 기준 (CLI 박제)
> 목적: 대회 생성 위저드의 종별·디비전·연령 체계를 디자인팀이 정확히 이해하고 화면을 설계할 수 있도록 현재 규칙을 정리.

---

## 1. 개요 — 3계층 구조

```
종별(Category)  >  디비전(Division)  >  [연령부(Age)]
   일반부              D3 ~ D8            (없음)
   유청소년            하모니·i1~i4       U8 ~ U18
```

| 계층 | 의미 | 출처 |
|------|------|------|
| **종별(Category)** | 가장 큰 분류 (일반/유소년/대학/시니어) | `AdminCategory` DB 마스터 (운영자 관리) |
| **디비전(Division)** | 종별 내 실력·체급 구분 (D3~D8, i1~i4, S1~S3 …) | 종별 마스터의 `divisions[]` |
| **연령부(Age)** | 유소년·시니어만 사용. 디비전과 곱집합으로 세부 종별 생성 | 종별 마스터의 `ages[]` |

> ⚠️ 종별/디비전/연령은 **코드에 하드코딩되어 있지 않고** 관리자가 `AdminCategory` 테이블에서 자유롭게 추가·수정합니다. 아래 표는 **현재 입력된 실제 값**입니다.

---

## 2. 종별 마스터 — 현재 4종 (실측)

| 정렬 | 종별 | 디비전 | 연령부 |
|:---:|------|--------|--------|
| 0 | **일반부** | D3 · D4 · D5 · D6 · D7 · D8 | — |
| 1 | **유청소년** | 하모니 · i1 · i2 · i3 · i4 | U8 · U9 · U10 · U11 · U12 · U13 · U14 · U15 · U16 · U17 · U18 |
| 2 | **대학부** | U1 · U2 · U3 | — |
| 3 | **시니어** | S1 · S2 · S3 | +40 · +45 · +50 · +55 · +60 · +65 · +70 |

- 디비전 `D3~D8 / i1~i4`: 숫자가 클수록 하위(입문) 쪽 실력 구분.
- 연령부 `U{N}`: N세 이하(유소년). `+{N}`: N세 이상(시니어).

---

## 3. 디비전 명명 규칙

| 규칙 | 설명 | 예시 |
|------|------|------|
| **여성 접미 `w`** | 여성부 선택 시 디비전명 뒤에 `w` 부착 | `D4` → `D4w`, `i3` → `i3w` |
| **종별 × 연령 결합** | 연령부가 있는 종별은 `디비전 연령` 형태로 결합 | `i3 U11`, `i2 U12`, `i3w U12` |
| **최종 종별명** | `성별 + 종별명` | "남성 일반부", "여성 유청소년" |

예: **여성 유청소년 i3w U12** = 여성부 / 유청소년 / i3 디비전 / 12세 이하

> 시스템 내부에는 운영자 정의 디비전(`divisions`)과 별개로 표준 티어(`division_tiers`, D1~D9)가 있으나, **위저드에서 운영자가 다루는 것은 `divisions`(위 표)** 입니다.

---

## 4. 연령설정 규칙

디비전별 연령 제한은 **출생연도 + 학년**으로 저장됩니다(`TournamentDivisionRule.birth_year_min/max`, `grade_min/max`).

### U{N} 매핑 패턴 (실측 — 대회연도 2026 기준)

| 연령부 | 출생연도 | 학년 상한 | 비고 |
|:---:|:---:|:---:|------|
| U9 | 2017년생 | 초3 | |
| U11 | 2015년생 | 초5 | |
| U12 | 2014년생 | 초6 | |
| U14 | 2012년생 | 중2 | |

**공식**
- 출생연도 = `대회연도 − N`  (예: 2026 − 12 = 2014년생 = U12)
- 학년 상한 = `N − 6`  (만 나이 기준 초1=만7세)
- 시니어 `+{N}` = N세 **이상**

> ⚠️ **자동 계산 아님** — 위저드는 종별명(예: "U12")만 받고, 실제 출생연도/학년 범위는 **대회 생성 후 종별·디비전 편집에서 운영자가 직접 입력**합니다. 위 매핑은 강남구 운영 사례에서 추출한 표준 패턴이며, 디자인 시 "연령 입력 보조(프리셋 버튼)" 제안의 근거로 활용 가능합니다.

## 5. 진행 방식 (Format) — 6종

| 코드 | 라벨(UI) | 의미 |
|------|---------|------|
| `single_elimination` | **토너먼트** | 싱글 엘리미네이션 |
| `round_robin` | **풀리그** | 전체 팀 리그전 |
| `dual_tournament` | **듀얼토너먼트** | 조별 더블 엘리미 |
| `group_stage_knockout` | **조별리그 + 토너먼트** | 조별 예선 → 본선 토너먼트 |
| `league_advancement` | **링크제** | 조별 풀리그 → 링크된 조끼리 순위전 |
| `group_stage_with_ranking` | **조별리그 + 동순위 순위결정전** | 조별 예선 → 동순위끼리 순위전 |

**현재 사용 분포(실측)**: 풀리그 3 · 조별+순위전 4 · 링크제 1 · 조별+토너먼트 1

## 6. Format별 추가 설정 (settings)

| 키 | 의미 | 적용 Format |
|------|------|-------------|
| `group_size` | 조당 팀 수 | 조 설정 Format 전체 |
| `group_count` | 조 수 | 〃 |
| `ranking_format` | 동순위전 방식 (`round_robin` \| `single_elimination`) | `group_stage_with_ranking` |
| `advance_per_group` | 조별 진출 팀 수 | `group_stage_knockout` · `dual_tournament` |
| `linkage_pairs` | 링크제 조 페어 `[[1,2],[3,4]]` | `league_advancement` |

**실측 예시**
| 종별 | format | settings | 해석 |
|------|--------|----------|------|
| i3 U11 | 조별+순위전 | `{group_size:3, group_count:2, ranking_format:single_elimination}` | 3팀 2개조 → 동순위 토너먼트 |
| i2 U11 | 풀리그 | `{group_size:5, group_count:1}` | 5팀 단일 풀리그 |
| i3 U9 | 링크제 | `{group_size:4, group_count:2, linkage_pairs:[[1,2],[3,4]]}` | 4팀 2개조 → A↔B 링크 순위전 |

## 7. 위저드 입력 흐름 (5단계)

```
Step 0  대회정보     대회명* · 정규대회여부* · 주최* · 주관* · 후원사 · 포스터
Step 1  일정·장소    경기날짜* · 코트 · 장소*
Step 2  종별·디비전 ⭐  ← 본 문서 핵심
Step 3  경기설정     유니폼색상 · 쿼터구성 · 파울/타임아웃 · 휴식시간
Step 4  접수·공개    게시기간 · 참가신청기간 · 결제 · 계좌
```

### Step 2 — 종별·디비전 (DivisionGenerator 모달, 4단계)

```
1단계 · 성별       [ 남성 | 여성 ]
2단계 · 종별 템플릿 AdminCategory 칩 선택 (일반부 / 유청소년 / 대학부 / 시니어)
3단계 · 디비전     선택 종별의 divisions[] 체크박스 (여성이면 자동 w 부착)
4단계 · 연령(옵션)  연령부 있는 종별만 노출 — ages[] 체크박스
        ↓
생성 결과: 디비전 × 연령 곱집합으로 카드 생성
        각 카드: [디비전명] · 정원(기본 16) · 참가비(기본 60,000원) · 경기날짜 · 코트
```

- 종별 1개 이상 필수 (없으면 제출 차단).
- 정원 기본 16팀 / 참가비 기본 60,000원 (카드별 수정 가능).

## 8. (참고) 경기 설정 — Format과 별개

진행방식(format)과 별도로 **경기 룰**(쿼터·파울·타임아웃)을 Step 3에서 설정합니다. 프리셋 4종 제공:

| 프리셋 | 구성 |
|--------|------|
| 7분 4쿼터 | 4Q · 7분 · 타임아웃 전1/후2 |
| 6분 4쿼터 | 4Q · 6분 · 타임아웃 전1/후1 |
| 10분 4쿼터 | 4Q · 10분 · 타임아웃 전2/후3 |
| 10분 전후반 | HALF · 10분 · 타임아웃 전1/후1 |

기본값: 10분 4쿼터 · 파울 5개 · 팀파울 보너스 5개 · 홈색 BDR Red(#E31B23) / 원정 BDR Blue(#0F5FCC).

## 부록 — 디자인 검토 포인트 (제안)

1. **연령 입력 보조**: 현재 운영자가 출생연도/학년을 수동 입력 → "U12 선택 시 2014년생·초6 자동 채움" 프리셋 버튼 제안(§4 공식 활용).
2. **종별×연령 곱집합 카드**: 유청소년 i3 × U10·U11·U12 선택 시 카드 3개 생성 → 일괄 설정(정원/참가비) UI 필요.
3. **여성 `w` 접미**: 시각적으로 성별 구분이 드러나도록(색/뱃지).
4. **Format별 settings 조건부 노출**: format에 따라 입력 필드가 달라짐(§6) → 동적 폼 설계 참고.
