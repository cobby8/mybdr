## 3. 검증 절차
프리뷰 https://mybdr-git-dev-mybdr.vercel.app 로그인 → `/admin`(대시보드)·`/admin/users`·`/admin/games` 등 `/admin/*` 화면. 셸은 전 `/admin/*` 공통이라 한두 화면이면 충분.

### 데스크톱 (창 넓게, ≥900px) — 기준: `01-백오피스.png`
- [ ] **우상단에 계정 메뉴 없음** (정본 정합). 헤더는 페이지 제목 + 우측 액션만.
- [ ] 좌측 사이드바: BDR 로고(`ts-sidebar__brand`) + nav 그룹 헤더 + 현재 메뉴 **active 강조**.
- [ ] 사이드바 **푸터에 계정 UserChip**(아바타=이니셜, 이름, 역할 라벨, chevron) + **로그아웃** + 테마 토글 + 마이페이지 + 사이트로 돌아가기.
- [ ] UserChip 클릭 → `/admin/me` 이동. **로그아웃 클릭 → 정상 로그아웃**(기능 보존 최우선).
- [ ] 카드/그리드 가로 스크롤·레이아웃 이탈 없음.

### 모바일 (창 좁게, ≤900px)
- [ ] 상단 **ts-topbar = 햄버거 + 현재 페이지 제목** 표시(상단 빈 공간 없음).
- [ ] 햄버거 탭 → **좌측 드로어 슬라이드 + 오버레이**.
- [ ] 드로어 푸터에도 **계정/로그아웃/테마/마이페이지/복귀** 전부 동작.

### 기능 회귀 체크 (가장 중요)
- [ ] 로그아웃이 데스크톱·모바일 **양쪽 모두** 접근·작동.
- [ ] 사이드바 메뉴 항목·권한별 노출·하위메뉴(BDR NEWS 등) 들여쓰기·링크 이동이 이전과 동일.

## 4. 합격 / 회귀 기준
- **합격**: 위 체크 전부 통과 + 정본 스크린샷과 시각 일치(색/간격/라운딩/사이드바 폭).
- **회귀(보고 필요)**: 계정/로그아웃 누락·접근 불가, 사이드바 깨짐(클래스 미적용으로 무스타일), active 미표시, 모바일 상단 빈 공간/햄버거 무동작, 메뉴 항목 누락.
