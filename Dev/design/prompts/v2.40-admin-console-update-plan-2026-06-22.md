## 2. v2.40 통합 콘솔 — 핵심 구조

### 2-1. IA: 1단독 + 4그룹 (19섹션)

```
(단독) 대시보드
운영          : 대회 관리 · 경기 관리 · 팀 관리 · 단체 관리 · 코트 관리
사용자·커뮤니티 : 유저 관리 · 커뮤니티 · 시즌 시상 · 신고 검토 · 건의사항
비즈니스      : 결제 · 요금제 · 광고 캠페인 · 파트너
시스템        : 분석 · 종별 관리 · 알림 · 활동 로그 · 시스템 설정
```

### 2-2. 통일 템플릿 (모든 목록 화면 동일 골격)

`PageHead`(eyebrow·아이콘·타이틀·부제·액션) → `StatRow`(요약 stat) → `Toolbar`(검색 + 상태 탭) → `DataTable`(정렬·상태 뱃지·행 68px) → `Drawer`(우측 상세). 이 동일 골격이 "통일감"의 핵심.

### 2-3. 신규 — 콘솔 내 드릴다운 상세 (라우트 `base:id(:tab)`)

`user:id`(개요·활동·기록·결제) / `tournament:id`(개요·참가팀·**대진표 트리**·**정산** 수입·지출·순이익) / `game:id`(스코어보드·라인업) / `team:id`(로스터·전적) / `org:id`(소속팀·운영진·승인) / `court:id`(편의시설·예약·승인) + `report:id` / `payment:id` / `campaign:id`(CTR KPI) / `partner:id` / `post:id` / `news:id`.

### 2-4. 신규 — 콘솔 내 생성·작성 플로우

`create-tournament`(5단계 위저드) / `compose-notification`(대상·채널·예약·미리보기) / `create-campaign` / `write-news` + 소형은 `AddModal`(유저 초대·경기/코트 등록·시상/종별/요금제/파트너 추가).

### 2-5. 디자인시스템

v2.39 Toss(`toss.css`) 유지 + `au.css` 통합 레이어 추가(헤더·통계·툴바·테이블·드로어). 등급 = **E**(AppNav 미적용 자체 셸, 디자인 토큰만 일치 — 사용자 영역 13룰의 AppNav frozen 룰은 미적용 대상).

### DB 실측 (schema.prisma 2026-06-22)

- ✅ 바인딩 가능: `organizations`/`organization_members`(단체) · `payments`(결제) · `notifications`(알림) · `ad_campaigns`(캠페인) · `partners`/`partner_members` · `suggestions`(건의) · `AdminCategory`(종별) · `UserSeasonStat`/`ShotZoneStat`(기록) · `AdminInboxState`(인박스 snooze — v2.32 "신규 후보"였으나 **이미 모델 존재**).
- ⚠️ **정산**(tournament:id 정산 탭) = 전용 settlement 모델 없음 → `payments` + `game_applications`(참가비) **집계 로직** 필요(신규 DB 0, 쿼리 신규).
- 🔴 **신규 DB 후보 1**: 커뮤니티 게시글 신고 = `game_reports`/`court_reports` 만 존재, **community_posts 신고 전용 모델 부재**. G6 post:id 신고 처리 = 모델 신설 or 해당 탭 "준비 중" 보류 결정 필요.

> ⚠️ 메모리 lesson(`lesson_sian_db_assumption`): 시안 'DB 미보유' 가정 ≠ 실측. 위 ⚠️🔴 2건 외엔 전부 기존 데이터 재사용 가능.
