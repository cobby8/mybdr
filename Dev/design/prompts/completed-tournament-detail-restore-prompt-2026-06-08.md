## 1. 컨텍스트

- **의뢰 대상**: `BDR-current/screens/TournamentCompleted.jsx` (종료 대회 화면)
- **운영 라우트**: `src/app/(web)/tournaments/[id]/page.tsx` 의 `status === 'completed'` 분기
- **우선순위**: **P0 (사용자 직접 회귀 신고)**
- **박제 등급 변화**: A → A (기능 복원 + 결산 영역 보존)
- **시급도**: 빠름 (운영 회귀)

### 1-1. 문제 (회귀)

2026-05-28 UB1 종료 화면 박제 시, 종료 대회는 **결산 5카드 화면으로 완전 교체**되고 기존 `TournamentDetail`의 탭(개요·일정·대진표·순위·참가팀)이 **완전히 제거**됨 (page.tsx 가 5카드 렌더 후 그 자리에서 `return`).

→ **그 이전 버전에서는 종료된 대회도 일정·대진표 등 모든 정보를 열람할 수 있었음.** 사용자가 "종료 대회의 상세 정보가 전혀 안 보인다"고 회귀 신고.

### 1-2. 핵심 사실

- **데이터는 DB에 그대로 존재** (matches·teams·standings 등). UI에서만 진입로가 막힌 것 → DB 이슈 아님, 디자인(화면 분기) 이슈.
- 일정·대진표·참가팀 탭은 기존 `TournamentDetail.jsx` 에 이미 존재 (클라이언트 API 패칭). 종료 대회에서도 동일 데이터 접근 가능.

## 2. 변경 요구사항

### 2-1. 핵심 변경

종료 대회 화면에서 **결산(5카드)과 대회 상세 정보(일정·대진표·참가팀)를 함께 열람**할 수 있게 한다. 즉 종료 후에도 "이 대회가 어떻게 진행됐는지" 전체 기록을 볼 수 있어야 함.

### 2-2. 디자인 방향 (Claude.ai 가 제안 — 아래 후보 중 택1 또는 더 나은 안)

종료 대회 한 화면에서 **결산 + 상세를 자연스럽게 공존**시키는 레이아웃을 제안할 것. 참고 후보:

- **후보 A (권장)**: 상단 = 결산 Hero + 5카드(현행 유지) / 하단 = "대회 기록" 섹션으로 일정·대진표·참가팀 탭 통합. 한 페이지 내 스크롤로 결산 → 상세 흐름.
- **후보 B**: 결산 5카드 화면에 "전체 정보 보기" 진입 → 기존 탭 화면(일정·대진표·참가팀) 노출 (탭/세그먼트 전환).
- **후보 C**: 상단 탭 자체에 "결산" 탭을 추가하고, 기존 개요·일정·대진표·순위·참가팀 탭과 동급으로 배치 (결산이 첫 탭, 기본 선택).

→ 어느 안이든 **결산 정보와 상세 정보 양쪽 모두 도달 가능**해야 함. 종료 대회 특성상 "결산"이 먼저 보이는 흐름 권장.

### 2-3. 제거 / 변경 금지 (보존 영역)

- **AppNav frozen** (03 코드 그대로)
- **결산 5카드** (최종순위 / MVP·베스트5 / 갤러리 / 알기자 스토리 / 다음회차) — 디자인·데이터 매핑 그대로 유지
- 종료 Hero (우승팀 / "종료된 대회" fallback)
- **데이터 패칭 0 변경** — 일정·대진표·참가팀은 기존 `TournamentDetail` 의 API/컴포넌트 재사용 (신규 쿼리·mock 금지)
- 신규 라우트 추가 ❌ — 동일 `[id]/page.tsx` 안 처리 (UB1 룰 답습)

## 3. 디자인 명세

### 3-1. 레이아웃 (후보 A 기준 예시)

```
데스크톱:
  [breadcrumb]
  [종료 Hero — 🏆 우승팀 / 대회 메타]
  [결산 5카드 grid]                    ← 현행 유지
  ─────────────────────────────────
  [섹션 제목: "대회 기록"]
  [탭바: 일정 | 대진표 | 참가팀]        ← 복원 (개요/순위는 결산과 중복 시 생략 검토)
  [선택 탭 콘텐츠]
  [Share bar]

모바일 (≤720px):
  1열 stack — Hero → 결산 5카드 → "대회 기록" 탭바(가로 스크롤) → 탭 콘텐츠
```

### 3-2. 컴포넌트

- 종료 Hero / 결산 카드: 기존 `tc-hero` / `tc-card` 시안 그대로
- "대회 기록" 탭바: 기존 `TournamentDetail` 의 탭 컴포넌트 톤 재사용
- 탭 콘텐츠(일정·대진표·참가팀): `TournamentDetail` 의 기존 패널 재사용

### 3-3. Interaction states

- **empty**: 일정/대진표/참가팀 데이터 없는 종료 대회 → 각 탭 "기록 없음" 안내 (mock ❌)
- **loading**: 탭 클라이언트 패칭 중 스켈레톤 (기존 패턴)
- 결산 카드: 데이터 없는 카드는 현행대로 자동 hide

### 3-4. Connections (페이지 간 흐름)

- 진입: /tournaments 목록 → 종료 대회 클릭 / 시리즈 페이지 → 과거 회차
- 탭 전환: 일정 ↔ 대진표 ↔ 참가팀 (한 화면 내)
- 다음 액션: 대진표 매치 클릭 → 경기 상세 / 다음 회차 카드 → 차기 대회
- 복귀: AppNav / breadcrumb
