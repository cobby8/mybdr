## Phase A4 — 드릴다운 상세 (G3)

> 콘솔 내 상세 페이지. 라우트 = `/admin/<base>/[id]` 신규 또는 Drawer→풀페이지. DB 확정분 우선.

```
[작업] v2.40 콘솔 드릴다운 상세 박제 — DetailHead→MiniStat→Tabs→2단/서브테이블

[배치 A4-1 — DB 확정 4] user · team · court · game
- user:id   → 개요 / 활동 타임라인 / 경기 기록 / 결제  (시안 au-detail.jsx AuUserDetail)
- team:id   → 개요 / 로스터 / 대회 전적                 (AuTeamDetail)
- court:id  → 개요(편의시설) / 예약 현황 / (대기=승인·반려)  (AuCourtDetail)
- game:id   → 개요(쿼터별 스코어보드) / 라인업 기록        (AuGameDetail)
- 데이터: 기존 모델 바인딩(User/TeamMember/courts·court_bookings/games·MatchPlayerStat 등). A0 매핑 참조

[배치 A4-2 — org + tournament(읽기요약, D2)] 
- org:id        → 개요 / 소속 팀 / 운영진 (대기 단체=승인·반려)  (AuOrgDetail · organizations/organization_members)
- tournament:id → 개요 / 참가팀(시드·입금) / 대진표 트리 / 정산  (AuTournamentDetail)
  ⚠️ D2: 콘솔 대회 상세 = site-operator **읽기 요약**. 편집·운영 액션은 tournament-admin/[id] 로 link 위임.
     정산 탭 = A0 집계 쿼리 결과(산출 가능 범위만, 보류분은 "—" 표기)
- 대진표 트리 = 읽기 전용 렌더(편집 X). 기존 tournament_bracket_versions 재사용

[배치 A4-3 — 엔티티 상세 (목록 행 클릭)] report · payment · campaign · partner · news
- report:id  → game_reports + court_reports 통합(D3: 커뮤니티 게시글 신고 = "준비 중" 탭)
- payment:id → 영수증·결제자·승인·환불 (payments)
- campaign:id→ 노출/클릭/CTR KPI·일별 차트·예산 (ad_campaigns/ad_placements)
- partner:id → 계약·연락처·활동 이력 (partners/partner_members)
- news:id    → 기사 상세 (news_photo 등). post:id 게시글 상세 = 본문·댓글(신고 강조)·검수
- 시안: au-detail2.jsx

[보존] tournament-admin 라우트·기능 미접촉 / 편집 권한 가드 / IDOR 소유자 검증
[검증] 각 상세 DB 실측 바인딩 확인(curl raw 1회) · 탭 전환 · 모바일 2단→1열 · tsc0
```
