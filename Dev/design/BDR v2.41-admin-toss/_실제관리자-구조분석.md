# 실제 관리자(/admin) 구조 분석 + 데이터 연결 (v2.42)

> 출처: `cobby8/mybdr@main` 운영 코드베이스 분석 → 백오피스 시안(`백오피스.html`)을 실 구조·실 데이터 형태로 정합.
> 목적: 시안이 **실제 운영 admin 과 1:1 매핑**되어, CLI 박제 시 라우트/필드/권한을 그대로 바인딩하도록.

## 1. 실제 admin 라우트 구조

- **라우트 그룹**: `src/app/(admin)/admin/*` — Next.js App Router. 진입 `/admin`.
- **레이아웃**: `src/app/(admin)/admin/layout.tsx`
  - `getAuthUser()` 인증 → 비로그인 시 로그인 리다이렉트(원경로 복귀).
  - `getAdminRoles(userId, session)` 로 권한 계산 → 역할 0개면 `/login?error=no_permission`.
  - `<AdminShell roles user topbarRight>` 로 wrap (= 시안의 셸 대응). 셸 source: `src/components/admin/admin-shell.tsx` + `sidebar.tsx` + `mobile-admin-nav.tsx`.
- **셸/네비**: `src/components/admin/sidebar.tsx` 의 `navStructure` 가 **단일 진실 소스**.

### 네비게이션 IA (navStructure — 1 단독 + 5 그룹)

| 그룹 | 항목 (라우트) | 권한(roles) |
|---|---|---|
| (단독) | 대시보드 `/admin` | all |
| **운영** | 대회 관리 `/admin/tournaments`¹ · 경기 관리 `/admin/games` · 팀 관리 `/admin/teams` · 단체 관리 `/admin/organizations` · 코트 관리 `/admin/courts` | super/site (+대회관리: tournament_admin) |
| **사용자·커뮤니티** | 유저 관리 `/admin/users` · 커뮤니티 `/admin/community`(→ BDR NEWS `/admin/news`) · 시즌 시상 `/admin/season-awards` · 신고 검토 `/admin/game-reports` · 건의사항 `/admin/suggestions` | super/site (시상·신고·건의 = super) |
| **비즈니스** | 결제 `/admin/payments` · 요금제 관리 `/admin/plans` · 광고 캠페인 `/admin/campaigns` · 파트너 관리 `/admin/partners` | super (캠페인: +partner_member) |
| **시스템** | 분석 `/admin/analytics` · 종별 관리 `/admin/categories` · 알림 `/admin/notifications` · 에이전트 `/admin/agents` · 활동 로그 `/admin/logs` · 시스템 설정 `/admin/settings` | super (분석: +site) |
| **외부 관리** | 협력업체 관리 `/partner-admin` | partner_member |
| (셸 푸터) | 마이페이지 `/admin/me` · 사이트로 돌아가기 `/` | — |

¹ `hrefByRole`: tournament_admin 은 `대회 관리` → `/tournament-admin/tournaments` 로 진입.

### 역할 매트릭스 (AdminRole)
`super_admin`(전체) · `site_admin`(운영+유저+커뮤니티+분석) · `tournament_admin`(대시보드+대회 관리) · `partner_member`(협력업체) · `org_member`.
필터 로직: `filterStructureByRoles()` — 항목/children 재귀 필터 + 빈 그룹 제거 + self차단·child노출 시 parent href rewrite.

### 아이콘
메뉴 정의는 Material Symbols 명, 렌더 시 `SIDEBAR_ICON` 맵으로 **lucide kebab** 변환(`dashboard→layout-dashboard`, `emoji_events→trophy`, `sports_basketball→volleyball`, `groups/group→users`, `location_on→map-pin`, `forum→message-square`, `report→flag`, `campaign→megaphone`, `grid_view→layout-grid`, `notifications→bell`, `list_alt→list`, `storefront→store` 등).

## 2. 데이터 소스

### 대시보드 (`admin/page.tsx`, `dynamic = force-dynamic`)
- **KPI 4** = Prisma count: `user.count()` / `tournament.count()` / `tournamentMatch.count({status in LIVE_MATCH_STATUSES})` / `team.count()`. (delta 없음 = 단순 카운트)
- **활동 추이** = `admin_logs` 최근 7일 일별 건수 (`$queryRaw GROUP BY TO_CHAR(created_at,'MM-DD')`), 0 채움.
- **최근 활동** = `admin_logs.findMany(take 5, orderBy created_at desc, include users)` → `action`(ACTION_LABEL 맵) · `description` · `severity`(info/success/warning/error) · 수행자 · 시각.

### 페이지별 (요약)
| 페이지 | 실 소스 |
|---|---|
| 유저/팀/단체/코트/대회/경기 | Prisma `user`/`team`/`organization`/`court`/`tournament`/`game` |
| 결제 | `payment` (status=`paid` + `paid_at`) |
| 요금제·캠페인·파트너 | `/api/admin/plans` · `/api/admin/campaigns` · `/api/admin/partners` (실 REST 라우트 존재) |
| 활동 로그 | `admin_logs` 전체 |
| 종별 | `admin/categories` 마스터 |
| 에이전트 | `admin/agents` (기록앱 동기화 · record-app-impact-panel) |

> 실재 admin API 라우트(`src/app/api/admin/*`): `campaigns`, `partners`, `plans`, `promo-stats`. 나머지 페이지는 서버 컴포넌트에서 Prisma 직접 조회.

## 3. 시안에 연결한 것 (백오피스 프로토타입)

| 항목 | 변경 |
|---|---|
| **NAV** | `bo-data.jsx > BO_NAV` = `navStructure` 1:1 (단독 대시보드 + 5그룹, 라우트 슬러그 id, roles, children=BDR NEWS, lucide 아이콘). |
| **역할 필터** | `admin-shell.jsx` 에 `roles` prop + `filterNav()`(운영 `filterStructureByRoles` 미러) 추가. 대시보드 상단 **역할 미리보기 스위처**(슈퍼/사이트/대회/협력업체)로 레일·패널 실시간 필터 시연. |
| **대시보드** | `admin/page.tsx` 형태로 교체 — 4 count KPI(유저/대회/진행중 경기/팀, delta 제거) + 활동추이 7일 막대 + admin_logs 최근활동(ACTION_LABEL + severity 점). |
| **페이지** | 실 라벨로 정합(대회/경기/팀/단체/코트 관리 · 유저 관리 · 커뮤니티 · 신고 검토 · 건의사항 · 결제 · 요금제 관리 · 광고 캠페인 · 파트너 관리 · 분석 · 종별 관리 · 알림 · 에이전트 · 활동 로그 · 시스템 설정). 신규 추가: **BDR NEWS · 시즌 시상 · 종별 관리 · 에이전트**. 각 블록에 실 Prisma/라우트 소스 주석. |
| **셸** | 듀얼네비 유지(아이콘 레일=그룹 + 컨텍스트 패널=항목). children=하위 들여쓰기(BDR NEWS), 단독 항목(대시보드)=자체 레일 섹션. |

> 모든 값은 실 데이터 형태를 본뜬 **mock**(정적 프로토타입 — 라이브 DB/세션 접근 불가). 박제 시 위 소스로 1:1 바인딩.
