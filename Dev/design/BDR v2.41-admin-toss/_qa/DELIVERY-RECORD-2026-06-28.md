## 구성 (시연 페이지)
- **백오피스(통합 셸)** — 관리자 홈 대시보드 + 운영 콘솔(유저/매칭/커뮤니티/코트/마케팅 인앱 콘솔, 대회/심판/협력업체 standalone) + 정산·플랜 + 시스템(알림·설정·마이페이지). 사이드바 스크롤·반응형 햄버거.
- **표준 콘솔** — 대회 콘솔(`대회 관리자.html`)·심판 콘솔(`심판 관리자.html`)·협력업체 콘솔. `관리자 홈.html`은 백오피스로 리다이렉트(폐기).
- **대회 워크스페이스** — 대회 생성/수정(사이드 5단계 + 종별 추가 4단계 모달 + 진행방식별 동적 settings + 대회 복사 + 대회 삭제), 대회 운영(참가팀·대진·일정·운영관리·사이트·정산).
- **공개 사이트** — `토너먼트 사이트.html?org=` 멀티테넌트(단체 소개·조직·공지·게시판·대회/정규대회 아카이브). MyBDR 앱 대회 상세 재설계.

## 종별·디비전 규칙 (반영 완료)
- 3계층: 종별 > 디비전 > [연령부]. 종별 4종: 일반부(D3~D8)·유청소년(하모니·i1~i4 / U8~U18 1년 단위)·대학부(U1~U3)·시니어(S1~S3 / +40~+70 5단위).
- 성별: 남성/여성(여성 디비전 'w' 접미). 종별×디비전×연령×진행방식 조합 선택.
- 연령 자동 채움: U{N}=대회연도−N년생 / 학년 N−6.
- 진행방식 6종 + 동적 settings(풀리그 rounds / 조별+토너 group·advance / 링크제 linkage_pairs / 조별+순위 ranking_format / 듀얼 advance / 토너먼트 없음).
- 대진·일정·경기 운영은 대회 운영 워크스페이스에서 처리(생성/수정 단계에서 대진 생성 제거).

## 구현 우선 순위(제안)
1. 인증/권한·역할 게이트 + 백오피스 데이터 API(목록·상세·상태 전환).
2. 대회 생성/수정 폼 저장 + 종별·진행방식·settings persist.
3. 대회 운영 워크스페이스 실 동작(대진·일정·정산·공지·발행).
4. 공개 사이트 발행 파이프라인(?org 멀티테넌트) + MyBDR 대회 상세 연동.
5. 매칭/코트/커뮤니티/마케팅 콘솔 실 데이터 + 액션.

## 미구현(시연으로 남김)
- 모든 저장·발송·승인·정산·결제·삭제 = 토스트/로컬. 실 API 연동 필요.
- 마케팅 콘솔: 캠페인 외 쿠폰·푸시 등 추가 기능.
- 진행방식 settings의 실제 대진 생성 로직(운영 워크스페이스).
