## 0. 왜 필요한가 (갭)
- 현 GameDetail 시안(v2.36)의 호스트 영역 = **출석 체크 + 참가자 리스트(표시만)**. 대기 신청자 **승인/거절·대기 순번·승격** UI 없음.
- 운영 `host-applications.tsx`는 status 0/1/2(대기/승인/거절) 승인·거절만. **M2 대기열(status 3·순번·승격) 미반영**.
- `/games/my-games` 하단 "내가 만든 경기"는 단순 목록 — 경기별 **신청현황·마감·확정·취소 관리** 부족.
- → 호스트가 "쉽게 개최하고 신청 관리·마감·확정"하는 흐름(원 기획 P1)이 미완성.

## 1. 컨텍스트

## 🔴 2-0. 반드시 선반영 — status 라벨 충돌 정리
운영에 **상충하는 status 맵 2개** 존재 → 시안은 **단일 출처(`game-status.ts`) 기준**으로 통일:
- `game-status.ts`(정): **1=모집중 / 2=확정 / 3=완료 / 4=취소**
- `my-games/page.tsx` 로컬맵(오): `3=진행중 / 4=완료 / 5=취소` ← **폐기 대상**. M1 박제 후 취소(4)가 "완료"로 오표시됨.
- 신청 status(game_applications): **0=신청완료(승인 대기) / 1=승인(확정) / 2=거절 / 3=대기(waitlist)** (+`waitlist_position`, `promotion_deadline`).
→ 시안의 모든 상태 라벨·뱃지 색은 위 정본만 사용. 박제 시 my-games 로컬맵 제거.

## 시안 E-1 — 호스트 신청·대기열 관리 (GameDetail 호스트 뷰)

### 2-1. 핵심 변경
기존 참가자 리스트(표시) 대신, 호스트 전용 **신청 관리 패널**로 확장. 3 구획:
1. **대기 승인 (status 0)** — 신청완료/승인 대기자. 행마다 [승인][거절]. 게스트는 GUEST 뱃지 + 경력 라벨(입문~선출) + 한마디 인용.
2. **확정 참가자 (status 1)** — 승인완료. 정원 `{승인수}/{max_participants}`. (출석 체크 섹션과 연결 — 기존 시안 C 유지)
3. **대기열 (status 3)** — 정원 마감 시 대기 신청자. **순번(waitlist_position) 표시** + 빈자리 발생 시 [승격] 또는 자동 알림 상태. 승격 시 `promotion_deadline` 카운트다운(표시용).

### 2-2. 동작
- 승인: status 0→1, `current_participants`++. 정원 도달 시 경기 status 1→2(확정) 자동(M1) — 패널 상단에 "정원 마감 · 확정" 표시.
- 거절: status 0→2.
- 승격(대기열): 빈자리 발생 → 대기 1번에게 알림(자동) + 호스트 화면에 "승격 알림 전송됨 · 응답 대기" 상태. **수동 강제 승격 버튼은 옵션**(정책=알림 후 수동). 무응답·만료 시 다음 순번.
- 권한: `me === organizer_id`만 노출(IDOR). 일반 참가자에겐 숨김.

### 2-3. 보존
- 기존 host-applications 승인/거절 동작·게스트 라벨·PlayerLink(닉네임→공개프로필). 출석 체크 섹션(시안 C). ApplyPanel(참가자 뷰). AppNav frozen.

### 3. 명세
- 레이아웃: GameDetail 본문에 "신청 관리" 카드(호스트만). 탭 또는 구획 3개(대기 승인 / 확정 / 대기열). 카운터 헤더(승인 N/정원 · 대기 N).
- 행: `40px 1fr auto`(아바타·정보·액션). 대기열 행은 좌측 순번 뱃지.
- States: 대기 0건/확정 0건/대기열 0건 각 빈 상태. 승격 알림 전송됨·응답대기·만료. 저장중. 정원 마감(확정).
- 모바일: 액션 버튼 full-width, 44px. 720px 분기.

## 시안 E-2 — 내 경기(호스트) 대시보드 (`/games/my-games` 강화)

### 2-1. 핵심 변경
`my-games` 하단 "내가 만든 경기"를 **호스트 운영 카드**로 강화(상단 참가자 신청내역은 보존):
- 경기 카드마다: 상태 뱃지(모집중/확정/완료/취소 — 정본 라벨) + **신청 현황**(승인 `{n}/{max}` · 대기 `{w}`) + 일시·장소 + **빠른 액션**.
- 빠른 액션: [신청 관리](→E-1) / [수정](/edit) / 상태별 [마감·확정](모집중·정원미달 수동 확정) / [취소](status 4, 확인 모달 → 신청자 알림 안내).
- 상단 요약 stat: 모집중 N / 확정 N / 완료 N (호스트 경기 기준).

### 2-3. 보존
- 상단 참가자 신청내역(confirmed/pending/completed/cancelled 4탭) 보존. 결제 경로 Link 유지. AppNav frozen.
- ⚠️ 단 4탭 상태에 **대기(waiting) 추가** — M2로 본인이 대기 신청한 경기도 노출.

### 3. 명세
- 레이아웃: 상단 참가자 내역(기존) → 하단 "내가 만든 경기"(호스트 카드 그리드). 데스크톱 2~3열 `auto-fill minmax`, 모바일 1열.
- States: 호스트 경기 0건 빈 상태("아직 만든 경기 없음" 5단어 이내 + [경기 만들기] CTA → /games/new). 취소 확인 모달.
- 취소 모달: "신청자에게 취소 알림이 갑니다" 안내(M1 GAME_CANCELLED 연동).
