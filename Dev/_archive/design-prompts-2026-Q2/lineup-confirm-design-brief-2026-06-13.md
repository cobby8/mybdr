## 1. 컨텍스트

- **의뢰 대상 페이지**: `/lineup-confirm/[matchId]` — 사전 라인업 확정 (팀장/매니저/운영자용)
- **우선순위**: **P1** (팀장이 경기 직전 모바일로 입력 — 사용 빈도 높음)
- **박제 등급 변화**: 시안 없음(자체 마크업) → **A** (신규 시안)
- **현재 상태**: 기능·DB 완전 동작. 시각 시안만 부재 — 기본 카드/체크박스 나열 수준
- **셸**: `(web)` 그룹 → **AppNav 적용** (03 frozen 카피). 본문만 신규 디자인
- **시급도**: STAGE A/B 와 병렬 가능 (독립)

## 2. 변경 요구사항

### 2-1. 핵심 변경
- 팀장이 **경기 전 출전 명단 + 주전 5명**을 빠르게 확정하는 화면. 현재 밋밋한 체크박스 나열을 **BDR v2 디자인 언어**로 재구성.
- **모바일 우선** — 팀장이 코트에서 폰으로 입력하는 상황 가정. 한 손 엄지 도달 + 44px 터치 + 하단 sticky 액션.

### 2-2. 신규/추가 요구
1. **매치 메타 헤더** — "라인업 확정" 타이틀 + 경기 일시 + 상태 뱃지(예정/준비/진행중/종료) + (운영자일 때) "운영자 권한" 칩
2. **우리팀 명단 보드** — 팀명 + 라이브 카운터 `출전 N명 · 주전 N/5`
3. **주전 5 슬롯 시각화 (권장)** — 주전 5명을 채워지는 5칸 슬롯 보드로 (게임 상세 `ParticipantsSlotBoard`/Concept B 패턴 재사용 검토). 채움=선수칩 / 빈칸=점선 placeholder. 한눈에 "5명 중 몇 명"이 보이게.
4. **선수 행(row)** — 출전 토글 / 주전 토글 / 등번호 / 이름 / 포지션. 출전↔주전 상하 관계가 시각적으로 드러나게(주전은 출전의 부분집합).
5. **단일 전체 토글 버튼** — 라벨 자동: 전원 출전이면 "전체 출전 해제" / 아니면 "전체 출전 선택"
6. **하단 액션** — "라인업 확정/재확정" (주력 CTA) + "라인업 해제"(기존 확정 있을 때만). 모바일은 **하단 고정 바**(MobileStickyBar 패턴) 권장 — 주전 진행도(N/5) + 확정 버튼 44px.

### 2-3. 보존 / 변경 금지
- **AppNav** 03 frozen 그대로
- **상대팀 영역 미노출** (사용자 결정 — 본인 팀만 표시. away 데이터 받아도 UI 0)
- **도메인 룰 보존**: 주전 ⊆ 출전 / 주전 정확히 5명 / 코치·매니저(role=coach/manager)는 출전 불가(후보 = role player·captain만)
- 데이터 패칭·API·라우트 변경 0 (UI만)

## 3. 디자인 명세

### 3-1. 레이아웃
- **데스크톱**: 단일 컬럼 max-w-3xl 중앙 정렬 (현행 폭 유지). 위→아래: 페이지헤더(홈으로 + match #) → 매치 메타 카드 → 주전 5슬롯 보드 → 선수 명단(전체 토글 + row 리스트) → 메시지 → 액션 버튼.
- **모바일(≤720px)**: 1열 stack. 액션 버튼은 **하단 sticky 바**로 분리(스크롤 무관 항상 노출). 선수 row는 터치 타깃 44px, 출전/주전 토글 좌측 정렬(엄지 도달).

### 3-2. 컴포넌트
- `LineupMetaCard` — 일시(schedule 아이콘)·상태 칩·운영자 칩
- `StarterSlotBoard` — 5칸(채움 선수칩 / 빈칸 점선). 4-1 카운터와 연동
- `PlayerLineupRow` — [출전 체크] [주전 토글] [등번호 칩] [이름(+닉)] [포지션 칩]. 코치/매니저 행은 출전 토글 disabled + "출전 불가" 회색 처리
- `AllToggleButton` — 단일 토글(라벨 자동)
- `LineupActionBar` — 확정/재확정 + 해제 (모바일 sticky)
- 아이콘 = Material Symbols Outlined (lucide ❌)

### 3-3. Interaction states
- **빈 명단**: "등록된 선수가 없습니다."
- **팀 미배정**: "아직 팀이 배정되지 않은 경기입니다. 대진 확정 후 다시 시도해주세요." (warning 톤 카드)
- **잠금(경기 시작 후, status≠scheduled/ready)**: "이미 시작된 경기입니다. 라인업을 변경할 수 없습니다." + 모든 입력 disabled
- **주전 5명 미달**: 확정 버튼 disabled + "주전 5명을 선택해야 확정할 수 있습니다. (현재 N/5)"
- **주전 5명 도달**: 미주전 선수의 주전 토글 disabled(추가 차단)
- **성공**: "라인업이 확정되었습니다." (success 톤)
- **에러**: route 검증 메시지 그대로 (error 톤)
- **처리중**: 확정 버튼 "처리중..."

### 3-4. Connections (페이지 간 흐름)
- **진입**: 대회 일정/내 경기 알림 → 라인업 확정 CTA / 직접 URL(`/lineup-confirm/[matchId]`)
- **권한 0**: 화면 미노출(서버 redirect "/")
- **확정 후**: 동일 페이지 갱신(재확정·해제 가능 상태로)
- **복귀**: 페이지헤더 "홈으로"(현행) — 진입점이 일정/알림이면 그쪽 복귀 라벨도 검토(가정 §6)

### 3-5. 데이터(실측 — 시안 라벨 정확히)
- 선수 1명 = 등번호(`jersey_number`, 없을 수 있음) / 이름(`user.name` 또는 `user.nickname` 또는 `player_name`) / 포지션(`position`) / 역할(`role`: player·captain·coach·manager)
- 카운터 = 출전(active = 주전+교체 합집합) / 주전(starters, 정확히 5)
- 상태값 = scheduled(예정)/ready(준비)/in_progress(진행 중)/completed(종료)

## 6. 질문 / 가정 (PM=수빈 결정 필요)
1. **주전 시각화**: 5슬롯 보드(권장) vs 단순 리스트 토글만. → 기본 가정 = **5슬롯 보드 채택**(한눈 파악). 반대 시 리스트만.
2. **모바일 하단 sticky 바**: 채택 권장(경기 직전 빠른 확정). → 기본 가정 = **채택**.
3. **복귀 동선**: 현행 "홈으로" 유지 vs 진입점(일정/알림) 복귀. → 기본 가정 = **"홈으로" 유지**(현행 동작 보존).
4. **포지션/역할 컬러칩**: 종별 컬러 언어(게임 탭) 차용 vs 중립 네이비. → 기본 가정 = **중립**(대회 맥락 — 픽업 종별색과 구분).
