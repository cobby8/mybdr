## 2. 🔴 백엔드 정합표 (실측 — 중복 생성 금지)
| 시안 state | mybdr 실제 | 조치 |
|------|------|------|
| 정규대회 isRegular/seriesId/seriesName | **`tournament_series` + `Tournament.series_id` 존재** | 재사용. **신규 정규대회 생성/목록 API만 보강**(시리즈 영역) |
| 장소·코트 venues[].{courtCount,naming} | **`courts`/`court_infos` 존재** | 재사용. 코트 수(1~8)·명명(숫자/알파벳)·다중코트 표현 정합 |
| dates[]↔courtIds[] (날짜별 코트 N:N) | **미존재** | ⚠️ **신규 관계 필요**(대회일자↔코트). schema diff 승인(§4) |
| divisions.{method,cap,fee,dateId,courtId} | `TournamentDivisionRule`(format/settings/feeKrw 존재) | Track B 확장: cap + **dateId/courtId 매칭** 추가 |
| 경기 설정(유니폼/방식/파울/타임아웃) | **`Tournament.game_rules` Json 존재** + game_time/ball/method | **기존 game_rules 형태에 채움**(신규 컬럼·테이블 금지). §3 게이트 |
| 게시 모달(결제수단/계좌) | `Tournament` bank/entry_fee 존재 | 재사용. PG '준비중'(계좌이체만) |
| 종별 마스터(admin_categories) | 신규(Track B) | Track B대로 생성+4종 시드 |
| 포스터/후원 이미지 | — | 업로드 스토리지 연결(mock=세션) |

→ **진짜 신규 = admin_categories + 날짜↔코트 배정 관계** 뿐. series/courts/game_rules/bank = **재사용**. raw SQL 금지, Prisma+BigInt.
