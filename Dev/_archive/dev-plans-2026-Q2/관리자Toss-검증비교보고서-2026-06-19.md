## 1. 결론 요약 (중요 — 진단 수정됨)
**관리자 화면은 "안 바뀐" 게 아니라 실제로 리스킨돼 있습니다.** 운영 admin은 라이트 배경·흰 카드·파란 액센트·뱃지 기반으로 이미 Toss 계열로 전환됐고, **다크 토글을 눌러도 라이트가 유지**됩니다(= Toss 라이트가 실제 적용됨).
그런데 "별로 안 바뀐 것 같다"고 느끼는 진짜 이유는 **토큰(색)만 Toss로 바뀌고, Toss 특유의 형태·여백 언어가 덜 입혀진 fidelity 격차** + **출발점이 이미 라이트한 깔끔한 admin이었던 점** 두 가지입니다. (이전 "셸이 opt-in 안 됨" 가설은 부차적 — 라이트 모드에선 프레임이 이미 라이트라 시각 변화가 작음.)

## 2. 현재 mybdr 관리자 상태 (직접 확인)
- **대시보드**: 라이트, 흰 KPI 카드 4(전체 유저 685 / 토너먼트 65 / 진행중 0 / 등록 팀 120), 파란 막대 차트, 최근 활동 리스트. 사이드바 흰색 + ADMIN 뱃지 + 다중 그룹 메뉴(콘텐츠/사용자/비즈니스).
- **유저 관리**: KPI 4(685/585/0/8, 숫자 의미색), 탭(전체/일반/호스트/관리자 + 카운트), 테이블(가입일시/닉네임/이메일/구독등급/운영권한/상태) — 셀에 뱃지칩, 라이트·깔끔.
- 다크 토글 → 라이트 유지(Toss 라이트 고정 동작).
→ **리스킨 적용됨. 깨지거나 옛 다크/레드로 남아있지 않음.**

## 3. BDR-join-v1 관리자 (기준) — 직접 확인
- **대시보드**: 흰 사이드바 4메뉴(대시보드/대회 관리/참가팀 관리/설정, 활성=연파랑 pill+파란 텍스트+lucide), KPI 4칸에 **소프트 컬러 아이콘 칩**(파랑/초록/주황 둥근 사각 아이콘 배경), 차트 카드, 큰 라운딩, 넉넉한 여백.
- **참가팀 관리**: pill 탭(전체 336/대기 0/취소 3), 큰 둥근 검색바 + 둥근 드롭다운(지역/대회), 테이블 셀에 **인라인 아이콘**(전화/위치핀/인원/트로피), 행 높이 크고 airy, 초록 "엑셀 다운로드" 버튼.

## 4. ⭐ 차이점 (mybdr 현재 ↔ BDR-join-v1) — 핵심
| 항목 | mybdr 관리자(현재) | BDR-join-v1(기준 Toss) | 격차 |
|------|------|------|------|
| 색/테마 | 라이트·파란 액센트 ✅ | 라이트·Toss Blue ✅ | **거의 일치** |
| 카드 라운딩 | 타이트(≈8~12px) | 큼(≈16~24px) | mybdr가 덜 둥글어 "Toss 느낌" 약함 |
| KPI 카드 | 라벨+큰 숫자(아이콘 칩 없음) | **소프트 컬러 아이콘 칩 + 라벨 + 숫자** | mybdr KPI가 밋밋 |
| 행 밀도 | 조밀 | airy(여백 큼) | mybdr가 빽빽 |
| 셀 장식 | 뱃지칩 위주 | **인라인 아이콘**(전화/위치/인원) | 정보 풍부함 차이 |
| 탭 스타일 | 언더라인 | pill | 스타일 불일치 |
| 검색/필터 | 컴팩트(우상단) | 큰 둥근 검색 + 둥근 드롭다운 인라인 | 형태 언어 차이 |
| 사이드바 | 다중 그룹·다수 메뉴(대형 플랫폼) | 4메뉴(단일 목적) | 스케일 차이(정당) |

→ **색은 맞췄으나 "둥근 카드·컬러 KPI 칩·여백·pill 탭·인라인 아이콘" 같은 Toss 시그니처가 덜 적용**돼, 전체적으로 "BDR-join-v1처럼 보이지 않고 깔끔해진 옛 admin"으로 읽힘.

## 5. 진단 (왜 "별로 안 바뀐 것 같은가")
1. **토큰 레벨 리스킨에 그침** — toss.css 색 토큰은 적용됐으나(`--color-*`→Toss 별칭), 컴포넌트의 **라운딩/여백/KPI 구조/탭/셀 아이콘**은 기존 mybdr 형태 유지. Toss의 형태 언어가 안 옮겨짐.
2. **출발점이 이미 라이트·깔끔** — mybdr admin은 Toss 전환 전에도 라이트 admin이라, 색 토큰만 바꾸면 시각 델타가 작음 → "변화가 약하다"는 체감.
3. (부차) 셸 프레임/다크 커버리지 — 내 수정 전엔 토픽바만 opt-in이라 **다크 모드**·일부 프레임에서 미전환이 있었으나, 라이트 모드 운영에선 영향이 작음.

## 6. 내 수정의 위치 (정직하게)
- `admin-shell.tsx`/`referee-shell.tsx` 셸 루트 `data-skin="toss"` 승격 = **다크 모드 강제 라이트화 + 사이드바/프레임 전체 커버리지 + 사문화된 `.admin-aside/.admin-table` override 활성화**에 유효.
- 단 **fidelity 격차(라운딩·KPI 칩·여백·탭·아이콘)는 셸 opt-in으로 안 풀림** — 컴포넌트 단위 재작업 필요.
- 따라서 내 수정은 "필요하지만 불충분". 배포 후 다크 모드에서 효과 확인 권장.

## 7. 권장 — Toss Fidelity Pass (CLI 재작업 항목)
"BDR-join-v1처럼" 만들려면 색 너머 형태까지:
1. **카드 라운딩 16~24px** 통일(`.ts-card`/admin 카드) — 현재 타이트 라운딩 상향.
2. **KPI 카드 = 소프트 컬러 아이콘 칩 + 라벨 + 숫자** 패턴으로 교체(대시보드·각 화면 상단 통계).
3. **행 높이/여백 상향**(airy) — 테이블 density 완화.
4. **탭 = pill 스타일**(전체/대기/취소 등) 통일.
5. **테이블 셀 인라인 아이콘**(전화/위치/인원 등) 적용 — 정보 가독성.
6. **검색/필터 = 큰 둥근 입력 + 둥근 드롭다운** 인라인 패턴.
7. (선행) 내 셸 루트 opt-in 수정 커밋·배포 → 다크 모드/프레임 전체 커버리지 확보 후 위 fidelity 적용.

## 8. 다음 액션
