# 새 대회 만들기 — 변경 계획 (BDR-join-v1 대비)

> 2026-06-19 · Cowork · 두 화면 직접 캡처 비교 (Claude in Chrome)
> 기준: **BDR-join-v1 `/admin/tournaments/create`** (사용자가 "깔끔·편하다"고 지목) · 대상: **mybdr `/tournament-admin/tournaments/new/wizard`**
> 연계: 대회 운영 리빌딩 Track B (`대회운영리빌딩-TrackB-CLI지시서-스키마대조-2026-06-19.md`) Phase 4. 본 문서가 "생성 폼" 부분 상세.

---

## 1. 직접 비교 (캡처 기반)

### BDR-join-v1 (기준 — 깔끔)
- **사이드바 1개**(Admin 4메뉴) + 본문. 군더더기 네비 없음.
- **단일 페이지 3컬럼**: ① 기본 정보(대회명·상태·성별·기간) / ② 경기장 정보(TMap 검색 or 직접) + 상세 정보 / ③ 종별 및 디비전 구성(+ 버튼으로 추가).
- **하단 고정 바**: [취소] + 큰 [대회 생성 완료]. 항상 보임.
- 우상단 **[이전 대회 불러오기]** = 빠른 시작.
- 모드 선택 없음. 한 화면에서 스캔·입력·생성 완결. 섹션 카드 + 아이콘 + 여백.

### mybdr 현재 (복잡)
- **이중 네비**: 좌측 admin 사이드바(전체 메뉴) **+** 상단 대회운영 서브탭(대회관리/대시보드/단체/내대회/시리즈/템플릿) → 가로 공간 잠식.
- **4개 모드 탭**: Quick(추천)·이름시작일만 / 단계별 3-step / PDF 요강 AI / 협회 대회 super admin → **시작 전 선택 부담**.
- 추가로 "협회 만들기" 프로모 박스.
- **좁은 단일 컬럼 + 긴 스크롤**: 필수 정보(이름·시작일·시리즈) → 아래로 계속 스크롤.
- 생성 후에도 **SetupHub 8~9 카드**로 종별·운영방식·대진 등 별도 설정 → "생성 ≠ 완결".

## 2. 핵심 문제 (왜 불편한가)
1. **네비 이중 적층** — admin 사이드바 + 서브탭이 동시에 떠 본문이 좁고 산만.
2. **모드 탭 4개 = 결정 피로** — 입력 시작 전에 경로부터 골라야 함.
3. **세로 1컬럼 긴 스크롤** — 전체를 한눈에 못 봄. BDR-join-v1은 3컬럼으로 한 화면.
4. **생성이 미완결** — 종별/디비전/운영방식이 생성 화면 밖(SetupHub)에 흩어져 단계가 늘어남.

## 3. 변경 계획 (BDR-join-v1 형태로)

### 3-1. 레이아웃 — 단일 페이지 3컬럼
- 모드 탭 제거. **기본 = 3컬럼 단일 폼**:
  - **Col 1 기본 정보**: 대회명(필수) · 상태 · 성별 · 시작/종료일 · (선택)소속 시리즈.
  - **Col 2 경기장 + 상세**: 경기장(직접 입력 + 검색) · 주최/주관/후원 · 경기시간/경기구/경기방식(기존 프리셋 입력 재사용) · 포스터/링크.
  - **Col 3 종별·디비전 구성**: [+]로 DivisionGenerator(종별 마스터 연동) → 종별 카드 + 디비전별 정원/참가비 인라인.
- **하단 고정 바**: [취소] + [대회 생성 완료]. 우상단 [이전 대회 불러오기].
- 모바일: 3컬럼 → 1열 stack(섹션 순서 유지), 하단 바 고정 유지.

### 3-2. 네비 정리 — 본문 집중
- 생성 화면에서 **상단 대회운영 서브탭 숨김 또는 축소** (focused create 레이아웃). admin 사이드바는 유지하되, 생성은 **풀폭 작업 모드**로(기존 `sidebarVariant="hidden"` wizard 풀스크린 패턴 활용 가능).
- 즉 "대회 만들기 = 한 화면 집중 작업"으로.

### 3-3. 모드 탭 → 보조 진입점으로 강등
- Quick의 본질(이름만 필수, 나머지 선택)은 **3컬럼 폼 자체가 이미 충족**(필수=대회명, 나머지 optional) → 별도 Quick 탭 불필요.
- **PDF 요강**·**협회 대회**는 탭이 아니라 **우상단 보조 버튼**([PDF로 채우기], [협회 마법사])으로. 메인 흐름을 단순화.

### 3-4. 생성 = 완결 (SetupHub 부담 축소)
- 종별·디비전을 **생성 화면(Col 3)에서 확정** → 생성 시 `TournamentDivisionRule` 자동 생성(Track B §4). 생성 후 SetupHub는 "남은 선택 항목"만(공개·기록자 등) 최소 표시.
- "이전 대회 불러오기"가 format/settings·종별까지 프리필(현 copy 한계 보완).

### 3-5. 디자인 — Toss 적용
- 섹션 카드(아이콘 헤더 + 둥근 16~24px), 입력 `.ts-input`, 하단 고정 바 큰 primary 버튼, 여백 넉넉. (관리자 Toss 디자인시스템 일관)

## 4. 적용 범위 / 의존
- 대상: `tournament-admin/tournaments/new/wizard` (+ `_v2` 생성 폼). Track B Phase 4 "대회 생성 폼(3컬럼)"의 구체화.
- 의존: 종별 마스터(admin_categories) + DivisionGenerator 연동(Track B §4) / 이전 대회 복사 보강.
- API/데이터 **변경 없음**(기존 Tournament 필드 + Track B 계약). 레이아웃·IA·디자인 재구성 위주.

## 5. 단계 (제안)
1. **3컬럼 단일 폼 레이아웃** + 하단 고정 바 + focused 네비(서브탭 숨김). 모드 탭 제거(PDF/협회 보조 버튼화).
2. **Col 3 종별·디비전 + DivisionGenerator(종별 마스터)** 통합 → 생성 시 DivisionRule 자동생성.
3. **이전 대회 불러오기** format/settings·종별 포함 프리필.
4. SetupHub 잔여 항목 최소화(생성 완결 후 "공개/기록자"만).
5. Toss 스타일 마감 + 다크/모바일 검증.
