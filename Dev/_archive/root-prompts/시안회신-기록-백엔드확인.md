## §6 확인 4건

**1) +/- (plus_minus) — ✅ 있음**
`MatchPlayerStat.plusMinus`(`@map("plus_minus")`, Int) 실재. 산출식 불필요. 경기=정수 / 집계=평균 / 팀=점수차 그대로 진행.

**2) 시즌 슛차트 — ✅ 데이터 있음(재도입 가능)**
`ShotZoneStat`(user_id·**season_year**·zone_code: RA/PAINT/MID_*…·attempts·made·efficiency) 실재. PM이 재도입 방식(탭/모달) 정하면 그대로 바인딩. 컴포넌트 유지해둔 것 적절.

**3) 팀 경기 박스 소스 — ⚠️ 정정: `TeamMatch` 모델 없음**
- `TeamMatch` 는 **존재하지 않음**(있는 건 `team_match_requests` = 친선 *신청*, 박스스코어 아님).
- **박스스코어는 `MatchPlayerStat` 이고, 오직 `tournament_match_id` 에만 연결됨**(game_id 없음). → **선수/팀 박스 기록은 "대회 경기"에서만 생성됨.** 친선·픽업 경기(`games`)는 박스스코어 없음.
- 올바른 조인 경로(팀 경기 기록):
  ```
  글로벌 Team.id
    → TournamentTeam.team_id (= Team.id)              [TournamentTeam.teamId 실재]
      → TournamentMatch (home/away_team_id = TournamentTeam.id)   = 팀의 "경기"
        → MatchPlayerStat (tournament_match_id = TournamentMatch.id,
                           tournament_team_player_id → TTP.tournament_team_id)
  ```
- **디자인 영향**: 시안의 팀/선수 "경기별" 탭은 **대회 경기만** 채워짐. 친선/픽업 게임을 같이 보여줄 의도였다면 → 빈 데이터(또는 "대회 경기" 라벨로 한정 표기 권장). 별도 친선 박스 도입은 신규(범위 밖).

**4) 링크 라우트 — 실제 경로 확정**
- 실재: `/users/[id]` · `/teams/[id]` · `/tournaments/[id]` (모두 존재).
- "기록/기록실"은 **각 상세의 탭**(현재 미존재 → 신규 탭). 제안: `/tournaments/[id]?tab=records` · `/users/[id]?tab=records` · `/teams/[id]?tab=records` (대회 탭은 `tournament-tabs.tsx` 패턴 = `?tab=` lazy 로드).
- 해시(`#/users/…`·`#/teams/…`) → 위 실제 라우트로 치환.

## 데이터 매핑 검증 (§4)
- 경기별/대회별/시즌별/팀 로스터/대회 기록실 → 전부 `MatchPlayerStat`(+ groupBy) 로 실현 ✅.
- "팀 경기 박스 = TeamMatch" 행만 → **위 3) 의 TournamentMatch 경로로 교체.**
