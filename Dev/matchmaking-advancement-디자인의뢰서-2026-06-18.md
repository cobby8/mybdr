# 시안 A — 경기 찾기/목록 UX 고도화 (M5)

## 1. 컨텍스트
- 대상: `Games.jsx` (운영 `/games`, 시안 `p2-ua1-games.html`)
- 우선순위 P2. 박제 등급: 기존 A → 디테일 보강.
- 기존 동작 유지(필터·맞춤필터·도시집계·페이지네이션) 위에 **UX 레이어만 추가**.

## 2. 변경 요구
### 2-1. 핵심
- **정렬 옵션** 드롭다운: 임박순(경기 시간 가까운)·모집임박순·최신순. (가까운순은 좌표 있을 때만 — 데이터 가용 시)
- **빠른 필터 칩** (목록 상단): 오늘 / 이번 주말 / 내 동네 / 모집임박 / 무료. 토글형, 다중 선택.
- **경기 카드 보강**: 모집 진행률 바(`current/max`) + 상태 뱃지("모집중"/"확정"/"마감임박"). 기존 `STATUS_LABEL` 색상 토큰 재사용.
- **빈 상태**: 결과 0건 시 안내 + "맞춤필터 끄기" / "인접 지역 보기" CTA.

### 2-3. 보존
- AppNav frozen / 기존 필터 사이드(또는 상단) 구조 / 카드 기본 레이아웃.

## 3. 명세
- 레이아웃: 데스크톱 = 정렬 드롭다운 우상단, 필터 칩 row는 목록 헤더 아래 가로 스크롤. 모바일 = 칩 가로 스크롤, 정렬은 아이콘 버튼+시트.
- 진행률 바: 카드 하단 얇은 바, 80%+ 시 강조색(토큰), 100% = "마감".
- States: 칩 active/inactive, 정렬 선택, 로딩 스켈레톤, 빈 상태.
- Connections: 카드 클릭 → `GameDetail`. 빈상태 CTA → 필터 리셋.

# 시안 B — 대기열(Waitlist) (M2)

## 1. 컨텍스트
- 대상: `GameDetail.jsx` 신청 패널 + `MyRegistrationStatus.jsx`(내 신청 현황) + `Games.jsx` 카드 뱃지
- 우선순위 P1. **확정 정책: 알림 후 수동 승격** (자동 아님).

## 2. 변경 요구
### 2-1. 핵심
- **GameDetail 신청 패널**: 정원 마감 시 "신청" → **"대기 신청"** 버튼으로 전환. 대기 신청 후 "대기 N번" 상태 표시.
- **대기 승격 모달/알림 CTA**: 빈자리 발생 시 대기 1번에게 "자리가 났어요" 알림 → 진입 시 **확정 CTA + 마감 카운트다운(예: 30분)** 모달.
- **MyRegistrationStatus**: 내 신청 목록에 "대기 N번" 상태 행 추가.
- **Games 카드**: 정원 마감 경기에 "대기 가능" 또는 "마감" 뱃지.

### 2-3. 보존
- 기존 신청/게스트 신청 패널 구조, 승인/대기 상태 표시 컴포넌트 톤.

## 3. 명세
- States: 모집중(신청) / 마감(대기신청) / 대기중(대기 N번) / 승격알림(확정 CTA) / 확정.
- 카운트다운: 단순 텍스트 타이머("00:29:50 남음" 류, 5단어 이내 라벨).
- Connections: 알림 → GameDetail 또는 전용 확정 모달. 무응답 → 다음 순번 안내.

# 시안 C — 출석 확정 + 리포트 출석/노쇼 구분 (M3)

## 1. 컨텍스트
- 대상: `GameDetail.jsx`(호스트 출석 체크 섹션) + 리포트 작성 폼(`GameResult.jsx` 또는 운영 `/games/[id]/report`)
- 우선순위 P1. 호스트 전용 기능.

## 2. 변경 요구
### 2-1. 핵심
- **호스트 출석 체크 섹션**: 경기 확정(status=2)~완료 시, 승인 참가자 리스트에 **출석 토글**(출석/미출석). 신규 패턴.
- **리포트 폼 보강**: 평가 대상에 **출석/노쇼 구분 표시**. 노쇼자는 신고 플래그(noshow, 기존)로 처리되도록 시각 구분.

### 2-3. 보존
- 기존 리포트 폼(전체평점·MVP·동행자평점·신고 플래그 5종) 구조 그대로. 출석 구분 레이어만 추가.

## 3. 명세
- 출석 토글: 체크박스 또는 토글 스위치(44px 터치), 행 = 아바타+닉네임+포지션+토글.
- States: 출석/미출석/저장중, 경기 전(비활성)·당일(활성).
- 호스트만 노출(권한). 일반 참가자에겐 출석 체크 UI 숨김.

# 시안 D — 평점 작성 유도 + 프로필 반영 (M4)

## 1. 컨텍스트
- 대상: `GameDetail.jsx`/`MyActivity.jsx` 평점 CTA 배너 + `Profile.jsx`(매너점수·MVP·참여수 노출)
- 우선순위 P1.

## 2. 변경 요구
### 2-1. 핵심
- **평점 CTA 배너**: 경기 종료 후 미작성 시 경기 상세/마이활동에 "평점 남기기" 배너. 작성 완료 시 사라짐.
- **프로필 '신뢰' 카드 신설**: HeroCard와 **별도**의 신뢰 카드에 **매너 등급 라벨 + MVP 횟수 + 참여 경기 수** 표시. (Phase 10 구현된 `manner_score` 기반)

### 2-3. 보존
- AppNav frozen / **HeroCard 그대로(레이팅·점수 미노출 — §3 보존)** / 기존 프로필 카드 구조. 신뢰 stat은 HeroCard가 아닌 별도 카드에만.

## ✅ PM(수빈) 결정 완료 (2026-06-18)
- **매너점수 노출 = 예. 형태 = 등급 라벨 / 위치 = 별도 '신뢰' 카드.**
- 매너점수는 숫자 대신 **등급 라벨**(예: 좋음/보통 류 — 구간 라벨은 시안에서 제안)로 표현. 숫자 직접 노출 안 함 → 점수 경쟁 압박·초기 데이터 희소 회피.
- HeroCard는 §3대로 **레이팅/점수 미노출 보존**. 신뢰 stat은 본문 별도 카드에만.
- → **룰 업데이트 필요**: `01-user-design-decisions.md §3` 계열의 "레이팅 노출 금지"를 "**매너점수(Phase 10 구현)는 등급 라벨로 별도 카드에 노출 허용. HeroCard 점수 노출은 여전히 금지**"로 갱신. (박제 시 CLI가 반영)

## 3. 명세
- CTA 배너: 카드 톤, "평점 남기기 →", 종료 경기 한정. 토큰 색상.
- 신뢰 카드: 매너 등급 라벨(구간→라벨 매핑 제안) + MVP N회 + 참여 N경기. 미평가 시 빈 상태("아직 기록 없음" 류 5단어 이내).
- 등급 구간(제안, 시안에서 확정): manner_score 기반 3~4단계 라벨. 색상은 토큰(--ok/--badge-* 계열).
